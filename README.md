# 慢病用药小管家 MVP

一个面向慢病患者的本地网页应用 MVP，用来把“问诊 - 购药 - 用药 - 续方”这条链路先串起来。

当前版本聚焦四个核心能力：

- 拍处方 / 拍药盒，自动识别建档
- 自定义每日提醒频次与具体时间点
- 拆分“待完成 / 已完成”的今日服药提醒
- 自动估算药量余量并提示复诊续方

## 当前版本做了什么

### 1. 拍照建档

支持上传处方图或药盒图片，系统会根据图片来源和文件名做一轮演示版识别，并自动预填：

- 药品名称
- 用途 / 适应症
- 单次剂量
- 每日频次
- 每次提醒时间
- 补货阈值与复诊周期

说明：当前是 MVP 演示版，采用前端规则识别和模板填充，重点是验证“图片建档”这条产品链路。

### 2. 每日频次对应具体提醒时间

用户不再只是选择“一天几次”，而是可以为每次服药单独设置具体时间，比如：

- 一天两次：08:00、20:00
- 一天三次：08:00、13:00、20:00

这更贴近真实的慢病服药场景，也能为后续消息提醒打基础。

### 3. 今日提醒分栏

今日服药提醒已经拆成两栏：

- 左侧：待完成
- 右侧：已完成

待完成支持：

- 标记“已服药”
- 设置“稍后提醒”

已完成支持：

- 撤销完成，重新回到待完成

### 4. 药量余量与续方提醒

系统会根据：

- 开始用药日期
- 当前库存
- 单次剂量
- 每日服药次数

自动估算剩余天数，并给出：

- 库存安全
- 待续购
- 高风险

同时根据上次复诊日期和复诊周期，推算下一次复诊节点。

## 运行方式

这个 MVP 是纯前端网页，不依赖后端服务。

### 方式一：直接打开

双击 `index.html`，浏览器即可打开。

### 方式二：启动本地静态服务

```powershell
cd E:\慢病用药小管家
python -m http.server 8080
```

然后访问：

`http://127.0.0.1:8080`

## MVP 说明

这个版本优先验证的是核心产品闭环，而不是复杂医疗能力，重点是：

- 能不能把“图片建档 -> 创建计划 -> 每日执行 -> 余量预警 -> 复诊提醒”跑通
- 能不能体现慢病用户“按时吃药、别断药、别错过复诊”的真实需求
- 能不能用最小成本做成一个可演示、可交互的网页 MVP

## 后续可以继续迭代的方向
